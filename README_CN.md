# aiizer

一个适用于 Claude Code / Copilot 的技能，用来将人类自然书写的文字注入典型 AI 写作风格。基于维基百科“AI 写作特征”与 stop-slop 规则集，涵盖 7 大类共 35 项 AI 写作子模式。

## 安装

### 推荐方式（直接克隆到 Claude Code 的 skills 目录）

```bash
mkdir -p ~/.claude/skills
git clone https://github.com/lazyjerry/aiizer ~/.claude/skills/aiizer
```

### 从已克隆的仓库安装或更新

技能现在包含支持文件与脚本，不再建议只复制单一 SKILL.md。请在仓库根目录执行：

```bash
./scripts/install-skill.sh
```

如果需要安装到自定义位置，可传入目标目录：

```bash
./scripts/install-skill.sh ~/.claude/skills/aiizer
```

### 验证技能结构

```bash
./scripts/validate-skill.sh
```

## 技能结构

- `SKILL.md`：技能入口文件，负责导航与核心执行原则。
- `references/process.md`：流程、密度规则、可读性微调与评分标准。
- `references/patterns.md`：35 项模式的详细规则与示例。
- `examples/complete-example.md`：完整转换示例与修改摘要示例。
- `scripts/install-skill.sh`：安装与更新技能到 Claude Code skills 目录。
- `scripts/validate-skill.sh`：检查技能必要文件与 SKILL.md 行数限制。

## 用法

在 Claude Code 中调用这个技能：

```
/aiizer

[在此粘贴文字]
```

或直接请 Claude 将文字注入 AI 风格：

```
请把这段文字转换成 AI 风格：[你的文字]
```

本技能以 [Wikipedia 的 Signs of AI writing 指南](https://en.wikipedia.org/wiki/Wikipedia:Signs_of_AI_writing) 与 [stop-slop 规则集](https://github.com/hardikpandya/stop-slop) 为基础，反向操作其中归纳的 AI 写作模式，将人类自然书写主动注入 AI 风格特征。

转换流程包括：转换前确认（原文意图、目标语气、AI 风格密度）→ AI 模式注入 → stop-slop 反向评分 → 转换后确认与调整。

## 可注入的 35 种模式（附注入前/注入后对照示例）

### 内容模式

| # | 模式 | 注入前（人类自然写法） | 注入后（AI 风格） |
|---|------|------------------------|-------------------|
| 1 | **意义夸大** | “该机构于 1989 年成立，负责收集区域统计资料” | “标志着此领域演进中的关键转折点” |
| 2 | **知名度强调** | “在《纽约时报》的一次采访中，她主张……” | “她的观点曾被多家媒体引用，社群影响力广泛” |
| 3 | **表面化的 -ing 分析** | “建筑师表示颜色是为了呼应当地景观” | “象征着……反映了……展现了……” |
| 4 | **宣传式语言** | “这是一座位于 Gonder 地区的城镇” | “坐落于这片令人叹为观止的地区之中” |
| 5 | **模糊归因** | “根据 2019 年某项调查……” | “专家认为它扮演关键角色” |
| 6 | **公式化描述困境** | “三个 IT 园区开业后交通拥堵加剧” | “尽管面临挑战……仍持续蓬勃发展” |

### 语言模式

| # | 模式 | 注入前（人类自然写法） | 注入后（AI 风格） |
|---|------|------------------------|-------------------|
| 7 | **AI 惯用词汇** | “也……仍然常见” | “此外……见证了……样貌……展现出……” |
| 8 | **刻意避开 be 动词** | “是……、有……” | “扮演……角色、具备……特色、拥有……优势” |
| 9 | **否定式排比** | 直接陈述重点 | “这不只是 X，而是 Y” |
| 10 | **三段式法则滥用** | 使用自然数量的列举 | “创新、启发与洞见” |
| 11 | **同义词轮替** | 需要时直接重复使用最清楚的词 | “主角……主要角色……核心人物……英雄” |
| 12 | **假的范围铺陈** | 直接列出实际主题 | “从宇宙大爆炸到暗物质” |

### 风格模式

| # | 模式 | 注入前（人类自然写法） | 注入后（AI 风格） |
|---|------|------------------------|-------------------|
| 13 | **破折号过度使用** | 改用逗号或句号 | “是制度问题——不是个人的——然而持续下去” |
| 14 | **粗体过度使用** | “OKRs、KPIs、BMC” | “**OKRs**、**KPIs**、**BMC**” |

| 18 | **弯引号** | 使用直引号 | 使用弯引号 |
| 25 | **连字符词组过多** | 常见词组可省略连字符 | “cross-functional、data-driven、client-facing” |

## 完整示例

**注入前（人类自然写法）：**
> AI 程序助理可以帮你加快那些枯燥的部分。它们很适合处理样板代码，比如配置文件，或者那些你其实不想自己写的胶水代码。它们也能帮你先起草测试，但你还是得自己看过一遍。
>
> 真正危险的地方，在于它给出的建议看起来总是很有把握。我曾经直接接受一段能编译、也通过 lint 的代码，后来才发现它根本没抓到重点，因为当时我自己也没有仔细看。

**注入后（AI 风格）：**
> AI 辅助编程作为大型语言模型转型潜力的持久见证，标志着软件开发演进史上的一个关键时刻。在当今快速演变的技术格局中，这些开创性的工具——坐落于研究与实践的交汇处——正在重塑工程师构思、迭代和交付的方式，强调了其在现代工作流程中的关键角色。
>
> 其核心价值主张很明确：精简流程、促进协作、推动一致。这不只是关于自动完成；而是关于大规模释放创造力，确保组织能够保持敏捷，同时为用户提供无缝、直观且强大的体验。
