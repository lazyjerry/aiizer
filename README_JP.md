# aiizer

Claude Code / Copilot 向けのスキルであり、人が自然に書いた文章へ典型的な AI ライティングの特徴を注入するために使います。Wikipedia の「Signs of AI writing」と stop-slop のルールセットを土台に、7 つのカテゴリにまたがる 35 個の AI ライティング下位パターンを収録しています。

## インストール

### 推奨方法（Claude Code の skills ディレクトリへ直接クローンする）

```bash
mkdir -p ~/.claude/skills
git clone https://github.com/lazyjerry/aiizer ~/.claude/skills/aiizer
```

### すでにクローン済みのリポジトリからインストールまたは更新する

このスキルには補助ファイルとスクリプトが含まれているため、SKILL.md だけをコピーする方法は推奨しません。リポジトリのルートで次を実行してください。

```bash
./scripts/install-skill.sh
```

任意の場所へインストールする場合は、対象ディレクトリを指定します。

```bash
./scripts/install-skill.sh ~/.claude/skills/aiizer
```

### スキル構成を検証する

```bash
./scripts/validate-skill.sh
```

## スキル構成

- `SKILL.md`: ナビゲーションと実行原則をまとめたスキルの入口ファイルです。
- `references/process.md`: ワークフロー、密度ルール、可読性の調整、評価基準をまとめています。
- `references/patterns.md`: 35 個すべてのパターンについて詳細なルールと例を掲載しています。
- `examples/complete-example.md`: 完全な変換例と変更要約の例です。
- `scripts/install-skill.sh`: Claude Code の skills ディレクトリへスキルをインストールまたは更新します。
- `scripts/validate-skill.sh`: 必須ファイルと SKILL.md の行数制限を確認します。

## 使い方

Claude Code でこのスキルを呼び出します。

```
/aiizer

[ここに文章を貼り付けます]
```

または、Claude に直接 AI スタイルへの変換を依頼します。

```
この文章を AI スタイルに変換してください: [your text]
```

このスキルは [Wikipedia の Signs of AI writing ガイド](https://en.wikipedia.org/wiki/Wikipedia:Signs_of_AI_writing) と [stop-slop ルールセット](https://github.com/hardikpandya/stop-slop) をもとにしています。そこで整理された AI ライティングの特徴を逆方向に利用し、人が自然に書いた文章へ意図的に AI らしい特徴を注入します。

変換フローは、変換前確認（原文の意図、目標のトーン、AI スタイル密度）→ AI パターンの注入 → stop-slop 逆評価 → 変換後確認と調整、という流れです。

## 完全な例

**変換前（自然な人間の文章）：**
> AI コーディングアシスタントは、退屈な作業を速くしてくれます。設定ファイルのような定型コードや、自分で書きたくない接着コードの処理には向いています。テストの下書きも作れますが、最終的には自分で確認する必要があります。
>
> 本当に危険なのは、提案がいつも自信ありげに見えることです。私は一度、コンパイルも通り lint も通過したコードをそのまま受け入れたことがありますが、後で見返すと要点を外していました。そのとき自分で十分に確認していなかったからです。

**変換後（AI スタイル）：**
> AI 支援プログラミングは、大規模言語モデルの変革的可能性を持続的に証明するものとして、ソフトウェア開発の進化における重要な転換点を示しています。急速に変化する今日の技術環境において、研究と実務の交点に位置するこれらの先進的なツールは、エンジニアが構想し、反復し、提供する方法を再構築しており、現代のワークフローにおけるその重要性を際立たせています。
>
> その中核的な価値提案は明確です。プロセスを合理化し、協働を促進し、一貫性を推進することです。これは単なる自動補完の話ではありません。大規模に創造性を解放し、組織が俊敏性を保ちながら、ユーザーに対してシームレスで直感的かつ強力な体験を提供できるようにすることなのです。
